The supplied tool's core dump behavior uses:
- 9600 8N2, DTR/DSR serial setup;
- SYST:RWL;
- DIAG:PEEK? -1,<0..255>,0;
- SYST:LOC;
- default output file caldump.bin.

This GUI reimplements the read-only dump protocol in Python with a graphical interface,
COM-port enumeration, progress, and checksum validation.

The v0.6 live CAL:STR write/verify workflow is separate application code based on the
user-tested standard SCPI CAL:SEC / CAL:STR command sequence. It does not implement
HPAK flash mode or any raw EEPROM write operation.
